Правило формирования идентификаторов для тестов.

Идентификатор теста (имя файла) состоит из 7 цифр, разделенных знаком подчеркивания:
1 цифра - тип теста (0 - CRUD, 1 - Regression, 2 - Positive, 3 - Negative, 4 - End-to-end),
2 цифра - логический модуль приложения (0 - запасная цифра, 1 - 1-й АРМ, 2 - 2-й АРМ и т.д.),
3, 4 цифры - подсистема модуля (00 - запасные цифры (если в АРМ-е нет подсистем), 10 - 1-я подсистема, 20 - 2-я подсистема и т.д.),
5, 6, 7 цифры - номер теста, 7-я цифра запасная, по умолчанию 0 (010 - 1-й тест, ..., 100 - 10-й тест и т.д.).
Дополнительно можно указать примечание, через знак подчеркивания (_name - проверка конкретной подсистемы).
